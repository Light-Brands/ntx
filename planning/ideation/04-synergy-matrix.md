# Synergy Matrix: Cross-Business Value Flows

> "Every connection is a channel. Value flows through all channels."

---

## Overview

This document maps the **value flows** between businesses in the Brand Factory ecosystem. Understanding these flows enables:
- Strategic launch sequencing
- Optimal resource allocation
- Identification of high-leverage opportunities
- Prevention of value leakage

---

## The Value Flow Framework

### Types of Value

| Value Type | Description | Examples |
|------------|-------------|----------|
| **Users** | People flowing between products | Creator → Platform → Community |
| **Data** | Information that improves other products | Usage patterns, preferences, outcomes |
| **Revenue** | Direct financial flows | Fees, subscriptions, token purchases |
| **Tokens** | Cryptographic value transfer | $LIGHT flowing between pillars |
| **Reputation** | Trust and credibility transfer | Success in one = credibility in another |
| **Infrastructure** | Shared technical capabilities | Identity, payments, agents |

### Flow Dynamics

```
┌─────────────────────────────────────────────────────────────┐
│                    VALUE FLOW DYNAMICS                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  DIRECT FLOWS                                                │
│  A ────────► B                                              │

│                                                              │
│  INDIRECT FLOWS                                              │

│  CIRCULAR FLOWS                                              │
│  A ───► B ───► C ───► A                                     │

│                                                              │
│  MULTIPLIER FLOWS                                            │
```

## Strategic Implications

### For Launch Sequencing

Based on synergy analysis, prioritize launches that:
1. Create infrastructure used by multiple pillars
2. Have strong bidirectional flows
3. Increase overall ecosystem connectivity
4. Don't create isolated value pools

### For Product Design

Design products to:
1. Use shared infrastructure (identity, tokens, data)
2. Create natural pathways to other products
3. Reward cross-product usage
4. Avoid recreating existing ecosystem capabilities

### For Token Design

Design tokens to:
1. Flow between pillars naturally
2. Increase utility with ecosystem participation
3. Create alignment across products
4. Avoid siloed token economies

### For Community Building

Build communities that:
1. Span multiple products
2. Create governance across pillars
3. Facilitate user-to-user connections
4. Support ecosystem-wide initiatives
