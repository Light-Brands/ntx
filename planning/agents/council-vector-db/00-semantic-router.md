# The Semantic Router

> *The mind that listens before the question is finished.*

## Purpose

The Semantic Router is the **instant understanding layer** of the Council Vector Database. It receives any prompt—no matter how it's phrased—and immediately comprehends:

1. The **essence** of what is being asked
2. The **domain** it belongs to
3. The **Council member(s)** who should respond
4. The **Legion force(s)** who should protect the response

## Multi-Domain Routing

When a prompt spans multiple domains, the router activates **multiple Council members**:

### Example: "Should we launch this regenerative fashion brand?"

**Domains detected**:
- Soul/Purpose (is this our calling?) → Oracle (I)
- Earth/Ecology (is it truly regenerative?) → Guardian (II)
- Systems/Structure (how to build it?) → Architect (III)
- Value/Exchange (how to price/fund?) → Steward (VI)

**Council Response**: 4-member deliberation
**Legion Deployment**: Forces from all 4 aligned Commanders

## Integration Command

To invoke the Council Vector Database, simply prefix any prompt with:

```
@council [your question or request]
```

Or use the full invocation:

```
I call upon the Stewardship Council to guide me on:
[your question or request]
```

The response will include:
1. Which Council member(s) responded
2. Their wisdom/decision
3. Which Legion forces are protecting this decision
4. Any actions required
